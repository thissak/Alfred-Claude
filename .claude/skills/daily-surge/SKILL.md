# 일별 전종목 이상패턴(Surge) 스크리닝

전 종목을 대상으로 당일 이상패턴을 탐지하고, 시장 등락률 보정 + 수급 매칭을 수행한다.

## 실행 절차

### Step 1: 인자 파싱

- 날짜 인자가 있으면 해당 날짜, 없으면 DB 최신 거래일을 사용한다.

### Step 2: 스크리너 실행

```bash
python3 scripts/daily_surge_screener.py [날짜]
```

스크립트가 자동으로 수행하는 작업:
- 시장 지수(KOSPI/KOSDAQ) 등락률 조회
- daily_screening 테이블에서 거래량 2.5x+ 또는 등락 7%+ 종목 1차 필터
- daily_prices에서 캔들 OHLCV 조회 (갭/장대봉/꼬리봉 탐지)
- investor_flow에서 당일 외인/기관 수급 매칭
- 시장 보정된 이상패턴 탐지 및 분류

출력: 터미널 리포트 + `data/daily_surge_{날짜}.json` 저장

### Step 3: 분석 리포트 제공

스크립트 출력을 바탕으로 사용자에게 인사이트를 제공한다:

인사이트 정리 기준:
- 시장 상황 요약 (KOSPI/KOSDAQ 등락률, 특이사항)
- 급등+거래량 종목: 가장 주목할 만한 종목, 수급 주체 분석
- 급락 종목: 하락 원인 추정 (악재, 이익실현, 시장 역행)
- 거래량 폭증(가격 변동 없음): 선행 시그널 가능성
- 캔들 패턴(위꼬리/아래꼬리): 매도/매수 압력 해석

### Step 4: 노션 저장

스크리닝 결과를 노션 "Daily Surge 스크리닝" 폴더에 개별 페이지로 저장한다.

- 부모 page_id: `336b83e1-2dd9-8141-b56f-c86bf06a33fb`
- 제목: "Daily Surge ({M/DD})"
- 아이콘: 🔥
- 내용: 시장 요약 + 패턴별 종목 테이블 (Notion `<table header-row="true">` 형식)
  - 급등+거래량, 급락, 거래량폭증(가격무변동), 캔들패턴 섹션별 정리
  - 각 종목: 종목명, 종가, 등락률, 거래량배수, 패턴, 외인/기관 수급

### Step 5: 관심 종목 심층 분석 (선택)

사용자가 특정 종목을 더 보고 싶으면 `/surge <종목코드>` 스킬로 개별 심층 분석을 연계한다.

## 탐지 기준 (시장 보정)

| 패턴 | 기준 |
|------|------|
| 급등 | 시장 대비 초과 +7% 이상 OR 절대 +20% |
| 급락 | 시장 대비 초과 -7% 이상 OR 절대 -20% |
| 거래량 폭증 | 5일 평균 대비 3배 이상 |
| 갭업 | 갭% > 0 이면서 시장 대비 초과 5% 이상 |
| 갭다운 | 갭% <= -3% (절대) 또는 시장 대비 초과 -8% |
| 장대양봉/음봉 | 몸통 70%+ 이고 시장 대비 초과 변동 7%+ |
| 꼬리봉 | 위/아래 꼬리 60% 이상 |

## 규칙
- 시장 대폭등/대폭락일에는 시장 보정이 핵심 — 개별 종목의 "진짜 이상" 움직임만 포착
- ETF/ETN은 1차 필터에서 제외되지만 레버리지 ETN 등은 포함될 수 있음
- 최소 시총 300억 이상만 대상
