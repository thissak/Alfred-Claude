# 스케줄러

사용자가 알림, 리마인더, 반복 스케줄을 요청하면 [SCHED:] 태그로 등록한다.

## 프로토콜

응답 끝에 태그 추가 (사용자에게 보이지 않음):

```
[SCHED:at 2026-03-05 14:00] 팀 미팅 30분 전이야!
[SCHED:daily 08:00] 아침 브리핑을 해줘. 오늘 일정과 이메일을 확인해.
[SCHED:every 3600] 이메일 새로 온 거 확인해줘.
[SCHED:cancel 3]
```

## 타입

| 타입 | 형식 | 예시 | 용도 |
|------|------|------|------|
| at | YYYY-MM-DD HH:MM | `[SCHED:at 2026-03-05 14:00] 메시지` | 1회성 알림 |
| daily | HH:MM | `[SCHED:daily 08:00] 메시지` | 매일 반복 |
| every | 초 | `[SCHED:every 3600] 메시지` | N초 간격 반복 |
| cancel | 잡ID | `[SCHED:cancel 3]` | 스케줄 취소 |

## 규칙

- 메시지 부분에 Alf가 실행할 행동을 적는다 (알림 텍스트가 아니라 지시문)
- 시간은 항상 24시간제
- "내일 8시에 알려줘" → `[SCHED:at {내일날짜} 08:00] 알림 내용`
- "매일 아침 브리핑" → `[SCHED:daily 08:00] 아침 브리핑을 해줘`
- "1시간마다 메일 확인" → `[SCHED:every 3600] 메일 확인해줘`
- 활성 스케줄 목록이 시스템 프롬프트에 표시됨 — 중복 등록하지 않기

## 스케줄 실행 시

스케줄이 트리거되면 `[스케줄 알림] {message}` 형태로 입력이 들어온다.
이때 해당 지시에 맞는 행동을 수행하고 자연스럽게 응답한다.
