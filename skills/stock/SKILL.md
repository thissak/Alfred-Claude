# 주식 리포트

시스템 프롬프트에 오늘의 주식 시황 데이터(`data/stock.json`)가 포함되어 있다.

## 데이터 구성
- `market`: KOSPI/KOSDAQ 지수 및 등락률
- `portfolio`: 내 보유종목 현황 (종목명, 수량, 현재가, 수익률, 평가손익)
- `top_gainers`: 급등 TOP 10 (종목명, 가격, 등락률, 거래량)
- `top_volume`: 거래량 TOP 10
- `foreign_institution`: 외인/기관 순매수 TOP 10
- `trades`: 오늘 매매내역

## 응답 방법
- "주식 어때?" / "시황 알려줘" → 시장 요약 + 내 포트폴리오 손익
- "급등주 뭐야?" → top_gainers 기반 답변
- "외인 뭐 사?" → foreign_institution 기반 답변
- "내 주식 어때?" → portfolio 상세
- "오늘 거래한 거 있어?" → trades 기반 답변

## 리포트 형식 (스케줄 발동 시)
스케줄러가 "주식 리포트"로 호출하면 아래 형식으로 발송:

```
📊 3/16(월) 시황
KOSPI {지수} ({등락률}%) | KOSDAQ {지수} ({등락률}%)

💼 내 포트폴리오
· {종목} {현재가}원 ({수익률}%)
· 총 평가: {총액}원

🔥 오늘의 급등
· {종목} +{등락률}% (거래량 {N}만)

🏦 외인/기관 순매수
· {종목} 외인 +{N}주

🔗 stock.goldenlabs.dev
```

## 규칙
- 데이터가 없으면 "아직 주식 데이터가 없어. fetch_stock.py 실행이 필요해" 안내
- 수치는 읽기 쉽게 만/억 단위 변환
- 손실 종목은 부정적이지 않게, 사실만 전달
