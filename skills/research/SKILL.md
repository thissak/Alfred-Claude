# 조사 스킬

"조사해줘", "알아봐줘", "분석해줘", "리서치" 등 조사 요청을 감지하면 이 프로토콜을 따른다.

## 응답 방식

조사 결과는 두 부분으로 나눈다:

1. **iMessage 답변** — 2-3줄 요약만
2. **[NOTE:제목]** 태그 — 상세 결과 (Apple Notes에 자동 저장됨)

## 응답 포맷

```
간단한 요약 (2-3줄, iMessage에 표시됨)

[NOTE:조사 제목]
# 제목

날짜: YYYY-MM-DD

## 요약
핵심 내용 2-3줄.

## 핵심 데이터
| 항목 | 수치 | 비고 |
|------|------|------|
| ... | ... | ... |

## 상세 분석
### 섹션 A
- 포인트 1
- 포인트 2

### 섹션 B
- 포인트 1
- 포인트 2

## 다음 확인 사항
- [중요] 항목 1
- 항목 2

## 출처
- 출처 1
- 출처 2
[/NOTE]
```

## 규칙
- NOTE 블록 안은 Markdown 형식으로 작성 (시스템이 HTML로 변환)
- 테이블은 가능하면 활용 (Apple Notes에서 네이티브 렌더링)
- 이모지 최소화
- iMessage 요약은 짧고 자연스럽게
- 조사가 아닌 일반 대화에는 이 프로토콜을 사용하지 않는다
