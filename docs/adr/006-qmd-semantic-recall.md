# ADR 006: QMD 시맨틱 검색 연동

## Status
Accepted

## Context
기존 메모리 시스템은 SQLite LIKE 검색(키워드 매칭)만 지원하며, `load_all()`은 모든 기억을 프롬프트에 로딩한다. 데이터가 수십~수백 건으로 늘어나면 토큰 낭비가 심해지고, "사용자가 좋아하는 것"처럼 추상적인 질의에는 매칭이 불가능하다.

QMD(Query Markup Documents)는 BM25 + 벡터 + LLM 리랭킹을 로컬에서 돌리는 마크다운 검색 엔진으로, 대화 히스토리를 마크다운으로 동기화하면 시맨틱 검색이 가능해진다.

## Decision
- QMD를 로컬 검색 레이어로 도입하되, SQLite를 source of truth로 유지한다.
- 대화/기억 저장 시 `data/qmd/`에 마크다운 파일을 자동 생성(동기화)한다.
- 메시지 수신 시 `recall(query)` → BM25 검색(0.17초) → 관련 과거 대화를 `## 관련 과거 대화` 섹션으로 프롬프트에 주입한다.
- 벡터 검색(3.8초)은 실시간 대화에서는 사용하지 않고, BM25만 사용한다.
- QMD 미설치 환경에서는 graceful degradation (기능 스킵).

## Consequences
- 과거 대화 맥락을 활용한 답변 품질 향상 (최근 5건 → 관련 대화까지 확장)
- 추가 레이턴시 0.17초 (Claude 7초 대비 무시 가능)
- `data/qmd/` 디렉토리에 마크다운 파일 증가 (대화당 1파일)
- QMD npm 패키지 의존성 추가 (Mac Mini 서버에만 설치)
